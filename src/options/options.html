__MSG_o_lGuideSymlink__

Saving outside the default download directory

The WebExtensions API does not allow files to be saved outside of the default download directory. A symlink inside the default download directory can be used to get around this limitation.

Invalid paths ( .. and /) are ignored. Files and directories with leading dots ( foo/.hidden) are considered invalid by Firefox and Chrome.


Windows 
mklink /D C:\path\to\symlink D:\path\to\actual
macOS/Unix 
ln -s /path/to/actual /path/to/symlink

Make sure the target directories actually exist, or the extension will silently fail to download.

Creating context menu items

To create a menu item to save cat.jpg into the directory ~/downloads/i/cats, with ~/downloads as the default download directory, add the line i/cats. 

    .
    i/dogs
    i/cats // cute!
    ---
    v/cats
    :date:/:day:/foo
    ---
    submenu
    >mammals
    >>i/cats // (alias: cute cats)
    >>i/dogs
    >i/others
Settings
➡
    ~/downloads/
    ~/downloads/i/dogs
    ~/downloads/i/cats
    ---
    ~/downloads/v/cats
    ~/downloads/2000-01-01/01/foo
    ---
    (submenu)
    (submenu)
    ~/downloads/i/cats (shows up as "cute cats")
    ~/downloads/i/dogs
    ~/downloads/i/others
Default download dir ~/downloads
➡

              


              
Comments and aliases


              

                To add comments, add // to the end of a line. For example,
                cats // very cute!.
              


              

                To change display names of menu items, add
                (alias: display name) to comments. For example,
                cats // very cute! (alias: cute cats). The menu item will then show
                "cute cats" instead of "cats".
              


              
To change the shortcut key, use (key: f) to manually assign one.


              
Submenus


              

                To create submenus, add >s to the line itself. An example can be
                seen above.
              


              

                Note: You can only save relative to the default download directory. See the next
                guide for a workaround.
