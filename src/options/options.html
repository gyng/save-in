__MSG_o_sGuideOutsideDownloadDirectory__

The WebExtensions API does not allow files to be saved outside of the default download directory. A symlink inside the default download directory can be used to get around this limitation.

Invalid paths ( .. and /) are ignored. Files and directories with leading dots ( foo/.hidden) are considered invalid by Firefox and Chrome.


Windows 
mklink /D C:\path\to\symlink D:\path\to\actual
macOS/Unix 
ln -s /path/to/actual /path/to/symlink

Make sure the target directories actually exist, or the extension will silently fail to download.

__MSG_o_sGuideCreatingMenuItems__

To create a menu item to save cat.jpg into the directory ~/downloads/i/cats, with ~/downloads as the default download directory, add the line i/cats. 

    .
    i/dogs
    i/cats // cute!
    ---
    v/cats
    :date:/:day:/foo
    ---
    submenu
    >mammals
    >>i/cats // (alias: cute cats)
    >>i/dogs
    >i/others
Settings
➡
    ~/downloads/
    ~/downloads/i/dogs
    ~/downloads/i/cats
    ---
    ~/downloads/v/cats
    ~/downloads/2000-01-01/01/foo
    ---
    (submenu)
    (submenu)
    ~/downloads/i/cats (shows up as "cute cats")
    ~/downloads/i/dogs
    ~/downloads/i/others
Default download dir ~/downloads
➡

          What is a WebExtension match pattern?
          

            It is a URL-shaped pattern, not a regular expression:
            <scheme>://<host>/<path>. Use * for HTTP or
            HTTPS, *.example.com for that domain and its subdomains, and
            /* for every path. For example, https://*.example.com/files/*.
            Write one complete pattern per line; do not add regex slashes, backslashes, or an
            explicit port.

__MSG_o_sGuideRenamingDownloads__

          

            Match downloads against
            regex patterns. When a download matches all patterns, rename it according to
            into:.
          


          
This renames all gif files to a UNIX timestamp.

          

            
fileext: gif
into: :unixdate:.:fileext:


            
=


            
cat.gif ➡ 1512136516.gif
 

          


          

            Ask an AI agent to draft routing rules
            

              Copy this prompt into an agent, or use WebMCP from External integrations.
            

            
Create Save In routing rules for this goal: [describe how files should be matched, renamed, and organized]. Use one clause per line and a blank line between rules. Every rule must include into:. Preserve :filename: unless I explicitly request renaming. Use only documented matchers and variables. Explain each rule, validate likely regex mistakes, then return a final copy-ready block.

          


          
__MSG_o_sGuideMovingDownloads__


          

            The following example moves all .gif files into the
            gifs subdirectory and renames the moved files to a timestamp.
          


          

            
fileext: gif
into: gifs/:unixdate:.:fileext:


            
=


            
cat.gif ➡ gifs/1512136516.gif
cat.jpg ✘ (no match)

          


          

            Routing is relative to the selected menu item. When you download a cat gif using the
            images context menu, the above example will save it as
            ~/downloads/images/gif/946684800.gif.
          


          

            End into: with a / to route into a folder
            without renaming — the file keeps its own name (the server's
            Content-Disposition/MIME name where available). For example
            into: gifs/ saves cat.gif as gifs/cat.gif.
          


          

            This advanced example renames downloads based off captured regex groups — the
            stuff inside () — and moves them into different directories.
            capture: filename selects the filename’s
            capture groups, and finally into: renames the files
            based off them using :$1:,
            :$2:, and :$3:.
          


          

            
// Move downloads based off versions
filename: (.*)\.(v.)\.(tar|gz)
pageurl: example.com
capture: filename
into: :$2:/:$1:.:$3:

            
=


            

ubuntu.v1.tar ➡ v1/ubuntu.tar
debian.v2.gz  ➡ v2/debian.gz
debian.zip    ✘ (no match)

          Where this works
          

            WebExtensions such as Foxy Gestures can message Save In directly. A Greasemonkey,
            Tampermonkey, or Violentmonkey userscript cannot do so by itself: userscripts do not

About Save In

          

            Save In is a local-first downloads helper for organizing files with context-menu
            destinations, routing rules, renaming, Page Sources, and download history.
          

          

            Save In has no analytics, advertising, developer-operated server, or remote executable
            code. Settings and history stay in browser-managed extension storage.
          

          
Why these permissions are needed

          

            
Context menus

            
Shows Save In commands on pages, links, media, and tabs.

            
Downloads

            
Starts saves, chooses filenames, monitors results, and supports retries.

            
Notifications

            
Reports completed saves and actionable failures when enabled.

            
Storage

            
Keeps settings, routing rules, local history, and restart-safe state.

            
Offscreen

            
Lets Chrome convert fetched file data into temporary download URLs.

            
Website access

            
Finds and saves resources the user chooses on arbitrary websites.
