You are writing a Downloads-menu definition for the Save In browser extension.

Create destinations for: images grouped by website, PDFs in documents, and a dated inbox.

Save In syntax:
- Write one menu instruction per line.
- A relative path such as images/cats creates a destination with that folder path.
- A line containing --- creates a separator.
- Prefix a child with >; use one additional > for each deeper submenu level.
- Add // (alias: Display name) to show a different menu label without changing the folder path.
- Add // comments after an instruction; add (key: f) inside the comment to assign an access key.
- Variables use :name: syntax. Useful choices include :sourcedomain:, :year:, :month:, :day:, and :filename:.
- A single . means the browser's Downloads folder.
- Paths must be relative. Never use an absolute path, .., or a leading slash.

Saving outside the Downloads folder

Browsers only let extensions save inside the default Downloads folder. To reach another location, create a symlink inside Downloads and use that relative symlink path as a destination. The target folder must already exist.

Paths containing .., absolute paths, and hidden path segments such as .private are not supported.

Windows 
mklink /D C:\Downloads\archive D:\Archive
macOS / Linux 
ln -s /path/to/archive ~/Downloads/archive

__MSG_o_lBrowserDownloadsHelp__

How do download URL filters work?
Enter one URL-like browser match pattern per line. It has a scheme, host, and path: <scheme>://<host>/<path>. For example, https://*.example.com/files/* matches HTTPS file URLs on example.com and its subdomains. Use *:// for HTTP and HTTPS, and end with /* to include every path. This is not a regular expression, and it cannot match query parameters or an explicit port.

__MSG_o_sRenameAndRouteDownloads__

Automatically move or rename downloads when their filename, source, page, or context matches.

How routing rules work
  • Write one clause per line.
  • Separate rules with a blank line.
  • Every matcher in a rule must match.
  • Every rule must end with an into: destination.
Move matching files and keep their names 
fileext: gif
into: gifs/
cat.gif becomes gifs/cat.gif.
Move and rename matching files 
fileext: gif
into: gifs/:unixdate:.:fileext:
cat.gif becomes gifs/1783929600.gif.

Destinations are relative to the Downloads-menu destination you selected. Matcher values are regular expressions; see the regular-expression guide.

Advanced: use capture groups

Parentheses capture parts of a matcher. Select the matcher with capture:, then use :$1:, :$2:, and later groups in into:. 

filename: (.*)\.(v[0-9]+)\.(tar|gz)
pageurl: example\.com
capture: filename
into: :$2:/:$1:.:$3:
ubuntu.v2.tar becomes v2/ubuntu.tar.
Ask an AI agent to draft routing rules

Copy this prompt into an agent, or use WebMCP from External integrations. Clause and variable reference 

You are writing Dynamic Downloads routing rules for the Save In browser extension.

Goal: [describe which downloads to match and where they should be saved]

Save In syntax:
- Write one clause per line and separate rules with a blank line.
- Every matcher in one rule must match. Matcher values are JavaScript regular expressions.
- Common matchers are filename:, fileext:, sourceurl:, sourcedomain:, pageurl:, pagedomain:, pagetitle:, context:, and mediatype:.
- Add /i to a matcher name for case-insensitive matching, for example filename/i: \.pdf$.
- Every rule must contain exactly one into: clause. Its value is a relative destination or filename template.
- End into: with / to move the file while preserving its resolved filename.
- Otherwise include :filename: unless the goal explicitly requests renaming.
- Variables use :name: syntax. Useful choices include :filename:, :fileext:, :sourcedomain:, :year:, :month:, :day:, :counter:, and :uuid:.
- To reuse regex groups, add capture: followed by the matcher name, then use :$1:, :$2:, and so on in into:.
- Lines beginning with // are comments. Never use absolute paths, .., or unsupported clause names.

If more syntax detail is needed, consult https://github.com/gyng/save-in/wiki/Clause-and-Variable-list before answering.

Explain each rule briefly, check for regex and path mistakes, then provide one final plain-text, copy-ready ruleset in a fenced code block.

          Where this works
          

            WebExtensions such as Foxy Gestures can message Save In directly. A Greasemonkey,
            Tampermonkey, or Violentmonkey userscript cannot do so by itself: userscripts do not
            receive cross-extension messaging access. Use a small companion WebExtension as a relay,
            or have the userscript hand the URL to an extension that exposes that ability.
          

          

            Save In does not adopt downloads already started by another extension, including Video
            DownloadHelper. The downloader must send Save In's DOWNLOAD message first,
            or a small relay extension must do so; Video DownloadHelper does not expose a stable
            integration API that Save In can call directly.

About Save In

          

            Save In is a local-first downloads helper for organizing files with context-menu
            destinations, routing rules, renaming, Page Sources, and download history.
          

          

            Save In has no analytics, advertising, developer-operated server, or remote executable
            code. Settings and history stay in browser-managed extension storage.
